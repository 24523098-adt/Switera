# Quick Task 260711-a8n: Briefing AI + Peta Overlay MIS + Notifikasi Target

## Objective

Menambah kompleksitas aplikasi (permintaan user) dengan tiga fitur yang tetap koheren dengan tema MIS:

1. **Briefing Harian AI** (Dashboard Manajer): naratif Gemini on-demand yang merangkum situasi hari
   ini, tindakan mendesak, kinerja vs target, dan prioritas — pola sama dengan Ringkasan AI Laporan.
2. **Peta Distribusi overlay MIS** (AnalisisRanking): PetaGeografis (Leaflet, selama ini hanya di
   Landing) diperluas dengan overlay kelas ABC + pemenuhan per kota; warna marker menurut kelas.
3. **Notifikasi target meleset**: sinkronNotifikasiMis juga membuat notifikasi `perhatian` bila
   tingkat pemenuhan / waktu kirim meleset dari target manajemen (dedupe 24 jam) — menutup loop MBO.

## Tasks

### Task 1: Backend — briefingService + route + notifikasi target

**Files:** `server/src/services/briefingService.js` (baru), `server/src/routes/misRoutes.js`,
`server/src/services/misService.js`

**Action:** `buatBriefingHarian()` mengagregasi getSituasiHariIni + getTindakanMendesak (top 5) +
getKpiManajer + getRekomendasiPrioritas (top 3) lalu memanggil geminiClient.generateText dengan prompt
briefing pagi (2-3 paragraf + prioritas bernomor, angka persis dari data). Route `POST
/mis/briefing-harian` (Manajer-only). sinkronNotifikasiMis ditambah blok target-meleset.

**Verify:** node --check; uji live endpoint via backend Docker (hot reload) dengan JWT manajer.

### Task 2: Frontend — kartu Briefing Harian + peta overlay

**Files:** `src/api/apiClient.js`, `src/pages/Dashboard.jsx`, `src/components/PetaGeografis.jsx`,
`src/pages/AnalisisRanking.jsx`

rekomendasi).
