# Quick Task 260710-sb7: Paket MIS — Kinerja vs Target, Tren, Drill-down

## Objective

Menjawab teguran dosen "arah MIS kurang jelas, masih condong TPS" dengan tiga ciri struktural MIS yang
belum ada:

1. **Target KPI (management by objectives)** — manajer menetapkan target; semua indikator menampilkan
   realisasi vs target, dan ambang eskalasi/stok tidak lagi hardcoded melainkan dibaca dari target.
2. **Riwayat KPI harian (time-series)** — snapshot KPI direkam otomatis satu baris per hari; grafik
   Tren Kinerja menjawab "apakah kinerja membaik dibanding periode lalu?".
3. **Drill-down KPI** — kartu KPI bisa diklik untuk melihat rincian pembentuk angkanya (memakai
   `computePemenuhanPerKota`/`computeSiklusDetail` dari `utils/mis.js` yang belum terpakai).

Catatan konteks: Dashboard Manajer saat ini belum memakai `/mis/kpi` sama sekali — paket ini sekaligus
menghadirkan bagian "Kinerja vs Target" baru yang menjadi konsumen endpoint itu.

## Tasks

### Task 1: Backend — model TargetKpi + KpiSnapshot, service, routes, seed

**Files:** `server/prisma/schema.prisma`, migration baru, `server/src/services/targetKpiService.js` (baru),
`server/src/services/misService.js`, `server/src/routes/misRoutes.js`, `server/src/schemas/misSchemas.js`
(baru), `server/prisma/seed.js`

**Action:**
- Model `TargetKpi` singleton (pola `Stok`): targetPemenuhan %, targetWaktuKirim hari, targetUtilisasi %,
  minHariPasokan, maxHariEskalasi — dengan default masuk akal.
- Model `KpiSnapshot`: satu baris per `tanggal` (unique), berisi tingkatPemenuhan, keputusanAktif,
  rataWaktuPengiriman, utilisasiKapasitas, stokTbs, totalPermintaanTon.
- `targetKpiService.js`: `getTargetKpi()` (fallback default bila belum ada baris) + `setTargetKpi()`
  (upsert + catatAktivitas, pola stokService).
- `misService.js`: `getKpiManajer` mengembalikan `{ ...kpi, target, status }` (tercapai/meleset per KPI);
  ambang di `getSituasiHariIni` (statusStok), `getTindakanMendesak`/`sinkronNotifikasiMis`
  (keputusan pending) membaca target, bukan konstanta; fungsi baru `rekamKpiSnapshotHarian()`
  (upsert by tanggal, dipanggil dari `sinkronNotifikasiMis` yang sudah dipanggil tiap dashboard dimuat)
  dan `getRiwayatKpi(hari)`.
- Routes (Manajer-only, pola misRoutes): `GET/PUT /mis/target-kpi` (PUT pakai zod `validate`),
  `GET /mis/riwayat-kpi`.
- `seed.js`: reset + seed TargetKpi default dan backfill 14 hari KpiSnapshot (data demo sintetis dengan
  tren, agar grafik bermakna saat presentasi) — konsisten dengan sifat seed demo yang sudah ada.
- Jalankan migrasi + backfill idempoten ke DB dev yang hidup, verifikasi endpoint live (login manajer).

**Verify:** `prisma migrate dev` sukses; GET /mis/target-kpi, PUT, GET /mis/riwayat-kpi, GET /mis/kpi
mengembalikan bentuk baru saat diuji live dengan JWT manajer.

### Task 2: Frontend — bagian "Kinerja vs Target" + "Tren Kinerja" + drill-down di Dashboard Manajer

**Files:** `src/api/apiClient.js`, `src/pages/Dashboard.jsx`

**Action:**
- apiClient: `getTargetKpi`, `setTargetKpi`, `getRiwayatKpi` (getKpi sudah ada → /mis/kpi).
- Bagian **Kinerja vs Target** setelah Situasi Hari Ini: kartu KPI (pemenuhan, waktu kirim, utilisasi,
  keputusan aktif) menampilkan realisasi, target, chip Tercapai/Meleset; tombol "Atur Target" membuka
  modal form (PUT lalu muat ulang); kartu bisa diklik → modal drill-down:
  - Pemenuhan → tabel `computePemenuhanPerKota` (per kota, % pemenuhan, terurut terendah).
  - Waktu kirim → tabel `computeSiklusDetail` (siklus terlama, tepat/terlambat vs ETA).
  - Utilisasi → tabel alokasi vs kapasitas per kota (hitung dari cache store).
- Bagian **Tren Kinerja**: line chart (useLiveChart) tingkat pemenuhan & utilisasi dari
  `/mis/riwayat-kpi`, gaya konsisten ProyeksiChart.
- Gaya Neo-Brutalism konsisten (border hitam 2px, hard shadow, token tokens.css).

**Verify:** `npm run build` sukses; endpoint live mengembalikan data yang cocok dengan tampilan.

## Catatan

Snapshot backfill di seed bersifat sintetis (14 hari tren) — dapat dipertahankan untuk demo; produksi
nyata akan terisi otomatis harian via sinkron-notifikasi.
